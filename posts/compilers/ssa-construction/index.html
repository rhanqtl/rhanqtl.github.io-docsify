构造 SSA 形式

Monday, March 13, 2023
Tags:
  • 编译器
  • 静态分析
  • SSA

  2 分钟阅读  

转换到 SSA 形式大体上分为两个步骤：

  1. 插入 phi 函数，确保对于任意的变量 v，USES(v) 中的每一个点有且仅有 v 的一个 reaching definition。这也称为“live-range splitting”；

  2. 重命名 v 的所有定义，给每个定义赋予唯一的名字（换言之，重命名 v 的所有 live-range）。

用 DF(n) 表示 n 的 dominance frontier，其中元素 x 为 n dominate x 的某个前驱顶点，但是不 strictly dominate x。

这是典型的 BFS 算法。

有几个注意点：

  • 由于 φ 节点本身是变量的定义，因此可能存在一种情况，即插入 φ 节点后又需要递归地处理被修改节点的 DF；

  • 此处使用 F 来避免在同一个 BB 中插入同一个变量的多个 φ 节点。这是因为同一变量不同 BB 的 DF 可能重合，而所有的情况最终都合并在同一个 φ 节点中处理，因此只要某一次处理某个 BB 的 DF 时插入了 φ 节点，后续就不需要再插入了。

Iterated Dominance Frontier

为了数学上的方便，先驱们给算法：插入 φ 节点（摘自 SSA Book） 计算过程中涉及到的所有 BB 进行了单独的定义，称为“iterated dominance frontier”，具体定义如下：

  1. DF1(S) = DF(S)

  2. DFi+1(S) = DF(UNION(S, DFi(S)))

最后达到的不动点用 DF+(S) 表示，该集合即为某个变量的“iterated dominance frontier”。实际上就是 DF(S) 的传递闭包。

例如 Dominance Frontier 例子（1） 中，DF+({B, C, D}) = {A, D, E}。

这样，上述算法就可以定义为在 DF+(S) 的起始处插入 φ 节点。

Join Set

对于给定的顶点集合 S，JOIN-SET(S) 是 S 中所有 join node 的集合。

所谓“join node”，是指能够从 S 中两个或多个不同的顶点经过不相交的路径到达的顶点。

实际上还有另一种更加直观地表示。我们先来进一步解读一下 DF 的定义。

这里需要注意循环的顺序：

  • φ-node 是在 BB 的起始部分插入的，当前 BB 可能用得到 φ 的结果，因此需要把处理 φ 节点的循环放在最前面

    • 这里其实存在一个小小的问题：如果先将 φ-node 的 LHS 重命名，后续在替换 RHS 的时候，是否会自己引用自己？答案是不会，φ-node 的 RHS 中的各个变量在处理 DT 中的父节点已经进行了替换，见最外层的第 3 个循环。这又引出一个问题：entry 没有父节点，怎么保证 φ-node 的 RHS 被处理呢？答案是，不需要处理，因为 entry 不可能有 φ-node

  • 处理完某 BB 的 φ-node 后，接下来就是处理普通的指令（这里称为“Statement”），这里嵌套的循环也同样需要注意顺序：先替换，再重命名，否则就有可能自己引用自己；
